# オレンジリーガル ウェブサイト 公開・運用手順書

運用担当メンバー向けの引き継ぎ資料です。本番公開までに必要な作業をまとめています。

- リポジトリ（ソースコード）: https://github.com/ShiraHide/OrangeLegalWebsite （Private）
- 確認用URL（レビュー用・Vercel）: https://orange-legal-website.vercel.app/
- 技術: Next.js（静的書き出し）＋ PHPメールフォーム

---

## 0. 構成の方針（決定事項）

| 項目 | 内容 |
|---|---|
| Webホスティング | **Xserver レンタルサーバー**（ドメインと同じXserverに集約） |
| サイト本体 | Next.js を**静的書き出し**（`out/` フォルダのHTML/CSS/JS/画像）してアップロード |
| お問い合わせフォーム | **PHP** で受け、**Google Workspace の SMTP 経由**でメール送信 |
| 第三者サービス | **使用しない**（Formspree等は不採用。機微情報を外部に通さない方針） |

> メールは既存の **Google Workspace** を利用するため、新たな外部サービスは増えません。
> 問い合わせ内容は「サイト → Xserver上のPHP → Google Workspace → 事務所の受信箱」という流れで届きます。

### コスト目安
- Xserver レンタルサーバー: 月額 **約1,000円**（契約期間により変動）
- 独自ドメイン: 取得済み（Xserver）
- Google Workspace: 既存
- 合計の新規追加コスト: **Xserverレンタル料のみ**

## 2. Google Workspace SMTP の準備

フォーム送信に使う Google アカウントで「アプリパスワード」を発行します。

1. 送信専用アカウントを用意（推奨）。例: `noreply@orangelegal.jp`
   （`cho@...` をそのまま使ってもよいですが、送信専用を分けると安全）
2. そのアカウントで **2段階認証を有効化**（Google アカウント → セキュリティ）。
3. **アプリパスワードを発行**（Google アカウント → セキュリティ → アプリパスワード）。
   - 16桁のパスワードが表示される → これを控える（通常のログインPWではない）。
4. 控えた情報（後で `contact.config.php` に設定）:
   - SMTPユーザー: `noreply@orangelegal.jp`
   - アプリパスワード: `xxxxxxxxxxxxxxxx`
   - 受信先: `cho@orangelegal.jp`

> 補足: Google Workspace 側でユーザー単位の送信が制限されている場合は、管理コンソールで該当アカウントの「安全性の低いアプリ」相当の設定や SMTP の利用可否を確認してください。

## 3. サイトを静的書き出しする（開発側の作業）

ソースコードから公開用ファイルを生成します。

```bash
npm install
npm run build      # → out/ フォルダに公開用ファイルが生成される
```

生成される `out/` の中身（これをアップロードします）:

```
out/
├─ index.html              … トップページ
├─ _next/                  … CSS/JS
├─ assets/                 … ロゴ・代表写真
├─ contact.php             … フォーム送信処理（PHP）
└─ contact.config.sample.php … 設定サンプル（後述）
```

---

## 4. PHPMailer を用意する

`contact.php` はメール送信に **PHPMailer**（無料ライブラリ）を使います。

1. https://github.com/PHPMailer/PHPMailer から最新版をダウンロード（「Code → Download ZIP」）。
2. 解凍してできた `PHPMailer-master` フォルダを **`PHPMailer`** にリネーム。
3. アップロード時に、サーバー上で次の配置になるようにします:
   ```
   （公開フォルダ）/lib/PHPMailer/src/PHPMailer.php
   （公開フォルダ）/lib/PHPMailer/src/SMTP.php
   （公開フォルダ）/lib/PHPMailer/src/Exception.php
   ```
   ※ `src` フォルダさえ上記パスにあればOKです。

---

## 5. Xserver にアップロード

1. Xserver のサーバーパネルで、対象ドメイン（`orangelegal.jp`）の**公開フォルダ**を確認
   （通常 `/home/アカウント名/orangelegal.jp/public_html/`）。
2. **`out/` の中身をすべて**この公開フォルダ直下にアップロード
   （`out` というフォルダごとではなく、**中身**を直下に置く）。
3. 手順4の **`lib/PHPMailer/`** も公開フォルダ直下にアップロード。
4. **`contact.config.php` を作成**（次の手順6）。

アップロード後のサーバー上の構成（イメージ）:
```
public_html/
├─ index.html
├─ _next/
├─ assets/
├─ contact.php
├─ contact.config.php     ← 手順6で作成（重要・要編集）
└─ lib/PHPMailer/src/...
```

---

## 6. 送信設定ファイル（contact.config.php）を作成

1. `contact.config.sample.php` を複製し、**`contact.config.php`** にリネーム。
2. 中身を実際の値に編集:

```php
<?php
return [
    'smtp_host' => 'smtp.gmail.com',
    'smtp_port' => 587,
    'smtp_user' => 'noreply@orangelegal.jp',   // 手順2のアカウント
    'smtp_pass' => 'xxxxxxxxxxxxxxxx',          // 手順2のアプリパスワード（16桁）
    'mail_from'      => 'noreply@orangelegal.jp',
    'mail_from_name' => '司法書士オレンジリーガル お問い合わせ',
    'mail_to'        => 'cho@orangelegal.jp',   // 問い合わせの受信先
];
```

> このファイルはパスワードを含むため、**サーバー上にのみ置きます**（Gitには登録しません）。

## 7. 独自ドメインと SSL の設定

1. Xserver サーバーパネルで `orangelegal.jp` をサーバーに追加（ドメイン設定）。
2. **無料独自SSL（Let's Encrypt）を有効化**（サーバーパネル → SSL設定）。
3. **DNS / ネームサーバー**:
   - ドメインをXserverで運用する場合、ネームサーバーをXserverに向ける、
     または該当のAレコードをXserverのサーバーIPに設定。
   - ⚠️ **メール（MX）はGoogle Workspaceのまま変更しない**こと。
     Webの向き先（A/CNAME）だけを変更します。
4. 反映には時間がかかる場合あり（最大48時間程度）。

## 8. 動作確認

1. `https://orangelegal.jp/` でサイトが表示されること。
2. スマートフォンでメニュー・下部の「ご相談はこちら」ボタンが出ること。
3. **お問い合わせフォームからテスト送信** → `cho@orangelegal.jp`（受信先）にメールが届くこと。

## 9. 公開後の更新フロー

サイト内容（文言・画像など）を直したいとき:

1. ソースを修正（開発側）。
2. `npm run build` で `out/` を再生成。
3. 変更分を Xserver にアップロード（再アップ）。

> 静的サイトのため「pushで自動デプロイ」はされません。更新のたびに再ビルド＆アップロードが必要です。
> （自動化したい場合は、GitHub Actions から FTP 自動アップロードを組むことも可能。希望時に対応します。）

## 10. レビュー段階の Vercel について

- 現在の `https://orange-legal-website.vercel.app/` は**メンバー確認（見た目レビュー）用**です。
- このVercel版は静的表示の確認用で、**フォームの実送信はXserver公開後に有効**になります
  （フォーム送信処理はXserver上のPHPで動くため）。
- 本番がXserverで公開できたら、Vercel側は停止して構いません。

## 12. よくある質問

**Q. 地図は表示されますか？**
A. 現状はデザイン上のダミーです。実際のGoogleマップ埋め込みが必要なら別途対応します。

**Q. フォームの内容はどこに届きますか？**
A. `contact.config.php` の `mail_to`（例: cho@orangelegal.jp）に、入力内容がメールで届きます。

**Q. プライバシーポリシー・特定商取引法の表記は？**
A. 現状未設置です。士業サイトでは個人情報の取扱方針の明記を推奨。必要に応じて追加できます。
