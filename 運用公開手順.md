## 2. サイトを静的書き出しする（開発側の作業）

ソースコードから公開用ファイルを生成します。

```bash
npm install
npm run build      # → out/ フォルダに公開用ファイルが生成される
```

生成される `out/` の中身（これをアップロードします）:

```
out/
├─ index.html      … トップページ
├─ _next/          … CSS/JS
└─ assets/         … ロゴ・代表写真・トップ画像
```

---

## 3. Xserver Static にアップロード

1. Xserver アカウントパネルで **Xserver Static** のサイトを作成。
2. 生成された **`out/` フォルダの中身**を、FTP/SFTP またはパネルのアップロード機能で公開領域へアップロード
   （`out` というフォルダごとではなく、**中身**を直下に置く）。

---

## 4. 独自ドメインと SSL の設定

1. Xserver Static のパネルで独自ドメイン（`orangelegal.jp`）を割り当て。
2. **無料SSL を有効化**。
3. **DNS**:
   - Web の向き先（A/CNAME等）を Xserver Static の指定どおりに設定。
   - ⚠️ **メール（MX）は Google Workspace のまま変更しない**こと（Webのレコードのみ変更）。
4. 反映には時間がかかる場合あり（最大48時間程度）。

---

## 5. 動作確認

1. `https://orangelegal.jp/` でサイトが表示されること。
2. スマートフォンでメニュー（ハンバーガー）が開くこと。
3. 各セクションへのスクロール、電話番号リンク（タップで発信）が機能すること。

---

## 6. 公開後の更新フロー

サイト内容（文言・画像など）を直したいとき:

1. ソースを修正（開発側）。
2. `npm run build` で `out/` を再生成。
3. 変更分を Xserver Static にアップロード（再アップロード）。

> 静的サイトのため「pushで自動デプロイ」はされません。更新のたびに再ビルド＆アップロードが必要です。
> （自動化したい場合は GitHub Actions から FTP 自動アップロードを組むことも可能。希望時に対応します。）
